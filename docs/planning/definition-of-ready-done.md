# Definition of Ready and Done

## Definition of Ready

An issue is ready when:

- It has a clear title.
- It has acceptance criteria.
- It has implementation tasks.
- Dependencies are known.
- Required design decisions are captured or linked.

## Definition of Done

An issue is done when:

- Code is implemented.
- Tests pass.
- CI passes.
- Documentation is updated.
- Security/privacy concerns are checked.
- No raw financial PII is logged or sent to AI.
