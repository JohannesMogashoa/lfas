# Planning

Planning data is source material for backlog management and GitHub portfolio
automation. It is separate from runtime code in `apps/` and `packages/`.

## Contents

- `backlog/` contains the source backlog spreadsheet exports.
- `portfolio/` contains issue bodies, labels, milestones, release definitions,
  and the portfolio model used by `scripts/portfolio`.

Use [Portfolio Roadmap](../docs/planning/portfolio-roadmap.md) for the product
sequence and [GitHub Bootstrap Notes](../docs/development/github-bootstrap.md)
for automation details.
